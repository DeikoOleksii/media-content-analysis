# Модель прецедентів

В цьому файлі необхідно перелічити всі документи, розроблені в проекті та дати посилання на них.

*Модель прецедентів повинна містити загальні оглядові діаграми та специфікації прецедентів.*


| **_ID:_**                | Сценарій 1 (реєстрація нового облікового запису)                               |
| ------------------------ | ------------------------------------------------------------------------------ |
| **_НАЗВА:_**             | Користувач хоче зареєструватись у системі                                      |
| **_УЧАСНИКИ:_**          | Користувач, Система                                                            |
| **_ПЕРЕДУМОВИ:_**        | Користувач не зареєстрований у системі                                         |
| **_РЕЗУЛЬТАТ:_**         | Новий обліковий запис                                                          |
| **_ВИКЛЮЧНІ СИТУАЦІЇ:_** | Системі було передано хибні дані для реєстрації                                |
| **_ОСНОВНИЙ СЦЕНАРІЙ:_** | 1. Користувач посилає системі запит на створення нового облікового запису      |
|                          | 2. Система перевіряє передані реєстраційні дані                                |
|                          | 3. Система створює новий обліковий запис, використовуючи введені дані          |
|                          | 4. Система надає користувачу інформацію про створення облікового запису        |
|                          | 5. Користувач завершує взаємодію з системою                                    |

| **_ID:_**                | Сценарій 2 (авторізація користувача)                                           |
| ------------------------ | ------------------------------------------------------------------------------ |
| **_НАЗВА:_**             | Користувач хоче війти в свій обліковий запис                                   |
| **_УЧАСНИКИ:_**          | Користувач, Система                                                            |
| **_ПЕРЕДУМОВИ:_**        | 1. Користувач не авторизований                                                 |
|                          | 2. У користувача є обліковий запис                                             |
| **_РЕЗУЛЬТАТ:_**         | Авторізація в системі                                                          |
| **_ВИКЛЮЧНІ СИТУАЦІЇ:_** | Системі було передано хибні дані для авторізації                               |
| **_ОСНОВНИЙ СЦЕНАРІЙ:_** | 1. Користувач посилає системі запит на вхід в існуючий обліковий запис         |
|                          | 2. Система перевіряє введені дані                                              |
|                          | 3. Система авторізує користувача                                               |
|                          | 4. Система повідомляє користувача і надає йому авторізацію                     |
|                          | 5. Користувач завершує взаємодію з системою                                    |

| **_ID:_**                | Сценарій 3 (формування запиту через надану системою форму)                     |
| ------------------------ | ------------------------------------------------------------------------------ |
| **_НАЗВА:_**             | Користувач хоче знайти інформацію за заданим шаблоном самостійно               |
| **_УЧАСНИКИ:_**          | Користувач, Система                                                            |
| **_ПЕРЕДУМОВИ:_**        | 1. Користувач авторизований в системі                                          |
|                          | 2. Користувач вміє користуватися системою і формою для запиту інформації       |
| **_РЕЗУЛЬТАТ:_**         | Результат аналізу медіа за даними, які ввів користувач                         |
| **_ВИКЛЮЧНІ СИТУАЦІЇ:_** | 1. Інформація за запитом користувача не була знайдена                          |
|                          | 2. Інформація, введена користувачем була введена некоректно                    |
| **_ОСНОВНИЙ СЦЕНАРІЙ:_** | 1. Користувач посилає системі запит на пошук даних                             |
|                          | 2. Система надає користувачу форму, за допомогою якої користувач може задати усі дані, за якими буде здійснюватися пошук |
|                          | 3. Система аналізує введену інформацію і починає пошук інформації              |
|                          | 4. Система надає користувачу результат пошуку, за заданими параметрами         |
|                          | 5. Система очікує на подальші дії клієнта, а саме на вибір користувачем фільтрівдля сортування результатів|
|                          | 6. Клієнт вибирає потрібний йому фільтр інформації                             |
|                          | 7. Система фільтрує дані за допомогою вибраного фільтра і знову надає користувачу результат|
|                          | 8. Клієнт оцінює якість сервісу                                                |
|                          | 9. Система надає користувачу можливість зберегти результат в будь-якому з наданих форматів|
|                          | 10. Користувач завершує взаємодію з системою                                   |
|                          | 11. Система зберігає запит в історію запитів                                   |

| **_ID:_**                | Сценарій 4 (формування запиту через адміністратора)                            |
| ------------------------ | ------------------------------------------------------------------------------ |

| **_УЧАСНИКИ:_**          | Користувач, Система, Адміністратор                                             |

|                          | 3. Система надає адміністратору форму, за допомогою якої адміністратор може задати усі дані, за якими буде здійснюватися пошук|

|                          | 8. Адміністратор передаэ результат користувачу                                 |
|                          | 9. Клієнт оцінює якість сервісу                                                |
|                          | 10. Система надає користувачу можливість зберегти результат в будь-якому з наданих форматів|
|                          | 11. Адміністратор завершує взаємодію з системою                                |
|                          | 11. Система зберігає запит в історію запитів                                   |
